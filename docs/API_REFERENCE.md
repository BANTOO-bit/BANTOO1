## 🛒 Order Domain

### `create_order`
Creates a new order with server-side price calculation and validation.

```sql
create_order(
    p_merchant_id UUID,
    p_items JSONB,              -- [{ menu_item_id, quantity, notes }]
    p_delivery_address TEXT,
    p_delivery_detail TEXT,
    p_customer_name TEXT,
    p_customer_phone TEXT,
    p_customer_lat FLOAT,
    p_customer_lng FLOAT,
    p_payment_method TEXT,      -- 'cod' | 'wallet'
    p_promo_code TEXT DEFAULT NULL,
    p_notes TEXT DEFAULT NULL
) RETURNS JSONB
```

**What it does:**
1. Calculates delivery fee server-side using tier config from `app_settings`
2. Calculates service fee from tier's admin_fee (not hardcoded)
3. Validates all menu items exist and calculates subtotal from DB prices
4. Validates promo code (if provided)
5. Deducts wallet balance (if wallet payment)
6. Creates order + order_items (order_number auto-generated by trigger)

**Returns:** Full order row as JSON

**Frontend:** `orderService.createOrder()`

### `update_order_status`
General status update for merchants, customers, and admins.

```sql
update_order_status(
    p_order_id UUID,
    p_status TEXT,          -- Target status
    p_additional JSONB DEFAULT '{}'  -- Extra data (cancellation_reason, prep_time, etc.)
) RETURNS JSONB
```

**Valid transitions:**
| From | To | Who |
|------|-----|-----|
| `pending` | `accepted` | Merchant |
| `accepted` | `preparing` | Merchant |
| `preparing` | `ready` | Merchant |
| `pending` | `cancelled` | Customer/Merchant |
| Any active | `cancelled` | Admin |

**Frontend:** `orderService.updateStatus()`

---

### `confirm_cod_payment`


## 🚗 Driver Domain

### `driver_accept_order`
Atomically assigns driver to order (uses `FOR UPDATE` lock to prevent races).

```sql
driver_accept_order(
    p_order_id UUID
) RETURNS JSONB     -- { success: bool, message: string }
```

**Validates:** Order is `ready`, no other driver assigned, driver is approved+active

**Frontend:** `driverService.acceptOrder()`

---

### `driver_reject_order`
Soft-rejects an order (also tracked in localStorage).

```sql
driver_reject_order(
    p_order_id UUID
) RETURNS JSONB
```

---

### `driver_update_order_status`
Updates order status from driver side with GPS validation.

```sql
driver_update_order_status(
    p_order_id UUID,
    p_status TEXT,      -- 'picked_up' | 'delivering' | 'delivered'
    p_lat FLOAT DEFAULT NULL,
    p_lng FLOAT DEFAULT NULL
) RETURNS JSONB
```

**Valid transitions:**
| From | To | Timestamp set |
|------|----|--------------|
| `pickup` | `picked_up` | `picked_up_at` |
| `picked_up` | `delivering` | — |
| `picked_up/delivering` | `delivered` | `delivered_at` |

**Frontend:** `driverService.updateOrderStatus()`

---

### `driver_complete_cod_delivery` ⭐ NEW
**Atomic** completion of COD delivery — single transaction instead of 2 RPCs.

```sql
driver_complete_cod_delivery(
    p_order_id UUID,
    p_lat FLOAT DEFAULT NULL,
    p_lng FLOAT DEFAULT NULL
) RETURNS JSONB
```

**Sets atomically:** `status='completed'`, `payment_status='paid'`, `delivered_at`, `completed_at`, `paid_at`

**Validates:** Order is COD, status is `picked_up/delivering`, driver owns order, not already paid

**Frontend:** `driverService.completeCodDelivery()`

## ⚙️ Utility Functions (Internal)

### `calculate_delivery_fee`
Tier-based delivery fee calculation from `app_settings`.

```sql
calculate_delivery_fee(p_merchant_id UUID, p_user_lat FLOAT, p_user_lng FLOAT)
RETURNS INTEGER  -- Fee in Rupiah
```

---

### `calculate_distance`
Haversine distance between two GPS points.

```sql
calculate_distance(lat1 FLOAT, lon1 FLOAT, lat2 FLOAT, lon2 FLOAT)
RETURNS FLOAT    -- Distance in km
```

---

### `get_service_fee_for_distance`
Returns admin_fee from delivery tier config for a given distance.

```sql
get_service_fee_for_distance(p_distance FLOAT)
RETURNS INTEGER  -- Fee in Rupiah (default: 2000)
```

---

### `generate_order_number`
Generates human-readable order ID: `BTN-YYMMDD-XXXX`.

```sql
generate_order_number()
RETURNS TEXT    -- e.g., 'BTN-260228-0042'
```

**Called:** Automatically by trigger on INSERT to orders table.

| `auto_close_inactive_merchants` | Cron | Close merchants without heartbeat |
| `auto_reassign_stale_driver_orders` | Cron | Reassign orders stuck with inactive drivers |
| `cleanup_ghost_drivers` | Cron | Set inactive drivers who haven't pinged offline |
| `auto_complete_old_delivered_orders` | Cron | Auto-complete delivered orders >24h |

## 📡 Realtime Channels

| Channel Pattern | Protocol | Used By | Purpose |
|----------------|----------|---------|---------|
| `driver-location-{orderId}` | Broadcast | Driver → Customer | Live GPS tracking |
| `tracking-order-{orderId}` | Postgres Changes | Orders table → Customer | Order status updates |
| `orders` | Postgres Changes | Orders table → Merchant | New/updated orders |
